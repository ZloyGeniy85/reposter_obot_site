# reposter_obot_site

Репозиторий разделён на три зоны: `frontend/` (лендинг, который видит пользователь), `api/` и `backend/` (заготовки под серверную часть — сейчас пустые, содержат только `.gitkeep`). Конфиги деплоя (`netlify.toml`, `render.yaml`) лежат в корне и указывают хостингу публиковать именно `frontend/`.

## Структура проекта

```
reposter_obot_site/
├── api/                     # заготовка под API (пока пусто)
│   └── .gitkeep
├── backend/                 # заготовка под серверную логику бота (пока пусто)
│   └── .gitkeep
├── frontend/                # сам лендинг — то, что публикуется на хостинг
│   ├── index.html
│   ├── assets/
│   │   ├── css/style.css    # Стили (вынесены из inline)
│   │   ├── js/
│   │   │   ├── i18n.js      # Логика переключения языков + localStorage
│   │   │   └── matrix.js    # Анимация фона (canvas)
│   │   └── img/             # favicon, og-cover и т.д. (положить сюда файлы)
│   └── locales/             # JSON-переводы: ru, en, ar, hi, fr, es
├── netlify.toml              # publish = "frontend"
├── render.yaml                # staticPublishPath = "./frontend"
└── README.md
```

**Про корневой `index.html`:** это не дубликат, а осознанный редирект-заглушка — через `<meta http-equiv="refresh">` он перенаправляет на `./frontend/index.html`. Сделано на случай, если хостинг по какой-то причине не подхватит `publish = "frontend"` из `netlify.toml` и будет отдавать корень репозитория как есть. Оба файла (`/index.html` и `/frontend/index.html`) нужно оставить — они выполняют разные роли, это не ошибка.

## Локализация (i18n)

- Поддерживаемые языки: `ru`, `en`, `ar`, `hi`, `fr`, `es`.
- Файлы переводов лежат в `/locales/<lang>.json`.
- Язык определяется в таком порядке:
  1. Сохранённый выбор пользователя (`localStorage.reposter_lang`);
  2. Язык браузера (`navigator.language`), если он входит в список поддерживаемых;
  3. Русский по умолчанию.
- Для арабского языка автоматически включается `dir="rtl"`.
- Чтобы добавить новый язык:
  1. Скопируйте `locales/en.json` → `locales/xx.json` и переведите значения.
  2. Добавьте `'xx'` в массив `SUPPORTED_LANGS` в `assets/js/i18n.js`.
  3. Добавьте кнопку `<button class="lang-btn" data-lang="xx">XX</button>` в `index.html`.

## Локальный запуск

Так как переводы подгружаются через `fetch()`, открытие `index.html` напрямую из файловой системы (`file://`) не будет работать из-за CORS. Запустите локальный сервер:

```bash
# Python
python3 -m http.server 8080

# или Node.js
npx serve .
```

Откройте `http://localhost:8080`.

## Деплой

### Netlify
1. Залейте репозиторий на GitHub.
2. На [app.netlify.com](https://app.netlify.com) → **Add new site → Import an existing project**.
3. Выберите репозиторий — Netlify подхватит `netlify.toml` автоматически (build command не нужен, `publish = "."`).
4. Домен вида `*.netlify.app` будет выдан автоматически, либо подключите свой.

### Render
1. На [dashboard.render.com](https://dashboard.render.com) → **New → Static Site**.
2. Подключите GitHub-репозиторий.
3. Build command оставьте пустым, Publish directory — `.` (или используйте `render.yaml` — Render подхватит его автоматически как Blueprint).

## Рекомендации по улучшению лендинга

См. раздел "Рекомендации" в описании задачи / PR — кратко:

1. **SEO**
   - Добавить `sitemap.xml` и `robots.txt`.
   - Реальные alt-тексты и `og:image` (сейчас плейсхолдер `assets/img/og-cover.png` — нужно заменить на реальное изображение 1200×630).
   - Для мультиязычности добавить `<link rel="alternate" hreflang="xx" href="...">` для каждого языка, если планируются отдельные URL (`/en/`, `/ar/` и т.д.) — сейчас переключение происходит через JS без смены URL, что для SEO хуже одноязычных URL-адресов на поддомене/подпути.
   - Ускорить LCP: сейчас anime-canvas рисуется поверх всего экрана и может отвлекать Lighthouse-метрики — рассмотреть `prefers-reduced-motion` media query.

2. **Динамика / UX**
   - Добавить реальные скриншоты/GIF работы бота (демонстрация переноса поста между каналами).
   - Добавить блок "Как это работает" (3 шага: добавить бота в донор → добавить в приёмники → настроить правила).
   - Добавить отзывы/кейсы (сколько каналов уже используют).
   - Добавить FAQ-аккордеон (частые вопросы: лимиты, безопасность аккаунта, поддержка).
   - Счётчик активных пользователей/каналов (даже статичный) добавляет доверия.

3. **Технически**
   - `prefers-reduced-motion` — отключать matrix-анимацию для пользователей с чувствительностью к движению.
   - Ленивая загрузка изображений (`loading="lazy"`), если добавите скриншоты.
   - Подключить Plausible/Umami (privacy-friendly аналитика) вместо тяжёлых трекеров.
